## 三、环境搭建

### 1. 工作目录选择

**关键点**：所有大文件都放在 `/root/autodl-tmp/`（数据盘 50GB），不要放在系统盘（30GB 不够用）。

```bash
cd /root/autodl-tmp
mkdir -p ft_cs && cd ft_cs
```

### 2. 安装系统依赖

```bash
apt-get update -qq && apt-get install -y -qq wget git vim
```

### 3. 安装 uv（替代 pip，速度快 10 倍）

```bash
curl -LsSf https://astral.sh/uv/install.sh | sh
source ~/.bashrc
```

### 4. 创建 Python 虚拟环境

```bash
uv python install 3.12
uv venv
source .venv/bin/activate
```

激活后提示符前会出现 `(ft_cs)`，确认环境激活。

### 5. 安装 PyTorch（坑点：源选择）

**❌ 错误尝试 1**：直接从 pytorch.org 下，速度只有 50KB/s，估计要 1-2 小时

```bash
uv pip install torch==2.5.1 torchvision==0.20.1 --index-url https://download.pytorch.org/whl/cu124
```

**❌ 错误尝试 2**：清华、阿里源都没有这个版本

```bash
# 这两个都失败
uv pip install ... --index-url https://mirrors.tuna.tsinghua.edu.cn/pytorch-wheels/cu124
uv pip install ... --index-url https://mirrors.aliyun.com/pytorch-wheels/cu124
```

**✅ 最终方案**：AutoDL 学术加速

```bash
source /etc/network_turbo
uv pip install torch==2.5.1 torchvision==0.20.1 --index-url https://download.pytorch.org/whl/cu124
```

速度 5-6 MB/s，3 分钟搞定。

> ⚠️ **学术加速注意事项**：
> - 只对当前 SSH 会话生效，断线重连要重新执行
> - 只对 github/huggingface 等学术域名加速，不是万能加速

### 6. 安装 LLaMA-Factory

```bash
uv pip install "llamafactory[torch]==0.9.2" accelerate peft transformers datasets trl --index-url https://pypi.doubanio.com/simple/
```

### 7. 跳过 vllm

**踩坑**：vllm 依赖 `nvidia-cudnn-frontend`，几个国内源都装不上。

**对策**：vllm 主要用于 API 推理加速，无卡模式下装了也没用。先跳过，需要时再装。

## 七、训练（踩坑最多的部分）

### ❌ 错误 1：用 screen 跑训练，退出 screen 把训练杀掉

```bash
# 错误做法
screen -S train
llamafactory-cli train ...
# 按 Ctrl+A D 想 detach，结果按错按了 exit
```

**结果**：训练在 step 137 时被中断，checkpoint 目录只有日志没有 LoRA 权重。

### ❌ 错误 2：缺少 `--do_train` 参数

```bash
# 错误命令（漏了 --do_train）
llamafactory-cli train \
    --stage sft \
    --model_name_or_path ./models/Qwen2.5-7B-Instruct \
    ...
```

**结果**：只跑了评估集 137 步就退出，没有真正训练。

### ❌ 错误 3：忘记加 `--report_to tensorboard`

第一次成功开始训练后，发现 TensorBoard 没有 events 文件，曲线看不到。

**修复**：加上 `--report_to tensorboard`，但需要先装 tensorboard 包：

```bash
uv pip install tensorboard --index-url https://pypi.doubanio.com/simple/
```

### ✅ 最终成功的训练命令

```bash
nohup bash -c 'source .venv/bin/activate && export HF_ENDPOINT=https://hf-mirror.com && export DATASET_DIR=$PWD/llamafactory && CUDA_VISIBLE_DEVICES=0 llamafactory-cli train \
    --stage sft \
    --do_train \
    --model_name_or_path ./models/Qwen2.5-7B-Instruct \
    --dataset ft_training \
    --eval_dataset ft_eval \
    --dataset_dir $DATASET_DIR \
    --template qwen \
    --finetuning_type lora \
    --lora_rank 16 \
    --lora_alpha 32 \
    --lora_target q_proj,k_proj,v_proj,o_proj \
    --output_dir ./checkpoints/ft_cs_v1 \
    --per_device_train_batch_size 4 \
    --per_device_eval_batch_size 4 \
    --gradient_accumulation_steps 4 \
    --learning_rate 2e-4 \
    --num_train_epochs 3 \
    --lr_scheduler_type cosine \
    --warmup_ratio 0.05 \
    --logging_steps 10 \
    --save_steps 200 \
    --eval_steps 200 \
    --evaluation_strategy steps \
    --preprocessing_num_workers 4 \
    --max_length 4096 \
    --ddp_timeout 180000 \
    --report_to tensorboard \
    --bf16' > train.log 2>&1 &
echo "训练已启动，PID: $!"
```

**关键点：**
- `nohup ... &` ：后台运行，SSH 断了也不影响
- `--do_train`：明确要训练（默认 False，会跳过训练）
- `--report_to tensorboard`：写入 TensorBoard 日志
- 不要用 `screen`，直接 nohup 更简单

## 十四、关键经验总结

### 选择对的工具
- `uv` 比 `pip` 快太多，强烈推荐
- AutoDL 学术加速对国内访问 GitHub/HuggingFace 是救星
- TensorBoard 看 loss 曲线比看 jsonl 直观

### 命令行的坑
- 中文参数要写到文件里读取，不要直接传
- 训练用 `nohup &` 而不是 `screen`，简单可靠
- 必须加 `--do_train`，否则不训练
- 必须加 `--report_to tensorboard`，否则没曲线

### 调试技巧
- 训练前先 `nvidia-smi` 确认 GPU 状态
- 训练中 `cat trainer_log.jsonl | tail -1` 看进度
- 训练完看 `eval_results.json` 判断效果
- Termius 中文乱码是显示问题，不是训练问题

### 资源管理
- 模型放数据盘（`/root/autodl-tmp/`），不要放系统盘
- 训练完立即备份 LoRA 权重到本机
- 不用时记得关机，按小时计费
